# MarketOverlay

**`MarketOverlay`**

Compound widget that combines market discovery with an overlay panel. Internally uses `useMarketFilters` to manage search, category, and sort state, renders a `MarketFilterBar` and `MarketList`, and opens a slide-over `Overlay` when a market is selected. The consumer provides a render prop that receives the selected `marketId` and returns the trading UI to display inside the panel.

```tsx
import { MarketOverlay } from '@functionspace/ui';
```

**CSS class:** `fs-market-overlay`

**Props:**

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `children` | `(marketId: number) => React.ReactNode` | required | Render prop receiving the selected market ID |
| `state` | `string?` | -- | Filter by resolution state (e.g., `'open'`) |
| `categories` | `string[]?` | -- | Fixed list of available categories |
| `pollInterval` | `number?` | -- | Polling interval in ms for refreshing market data |
| `emptyMessage` | `string` | `'No markets found'` | Custom message when no markets match |
| `showFilterBar` | `boolean` | `true` | Show or hide the `MarketFilterBar` |
| `featuredCategories` | `string[]?` | -- | Categories to display first in the filter chip row |
| `sortOptions` | `SortOption[]?` | 5 defaults | Custom sort options for the filter bar |
| `searchPlaceholder` | `string?` | `'Search markets...'` | Placeholder text for the search input |
| `filterBarMaxWidth` | `string?` | -- | CSS max-width for the filter bar |

**How It Works:**

1. When `showFilterBar` is `true` (default), `MarketOverlay` calls `useMarketFilters` internally with the provided props (`state`, `categories`, `featuredCategories`, `sortOptions`, `pollInterval`).
2. It renders a `MarketFilterBar` (driven by `filterBarProps` from the hook) above a `MarketList`.
3. When the user clicks a market card, the overlay panel slides open with the market title.
4. The `children` render prop is called with the selected `marketId`, and the returned elements render inside the panel.
5. Closing the panel resets selection back to the market list.
6. When `showFilterBar` is `false`, `MarketOverlay` falls back to `useMarkets` directly (no filter UI).

**Example: Simple overlay with filter bar**

```tsx
import { FunctionSpaceProvider } from '@functionspace/react';
import { MarketOverlay, MarketCharts, TradePanel } from '@functionspace/ui';

function App() {
  return (
    <FunctionSpaceProvider config={config} theme="fs-dark">
      <MarketOverlay state="open">
        {(marketId) => (
          <>
            <MarketCharts marketId={marketId} />
            <TradePanel marketId={marketId} />
          </>
        )}
      </MarketOverlay>
    </FunctionSpaceProvider>
  );
}
```

**Example: With featured categories and custom sort**

```tsx
import { FunctionSpaceProvider } from '@functionspace/react';
import { MarketOverlay, MarketCharts, TradePanel, PositionTable } from '@functionspace/ui';

function App() {
  return (
    <FunctionSpaceProvider config={config} theme="fs-dark">
      <MarketOverlay
        state="open"
        featuredCategories={['sports', 'crypto', 'politics']}
        sortOptions={[
          { field: 'totalVolume', label: 'Most Active', defaultOrder: 'desc' },
          { field: 'expiresAt', label: 'Closing Soon', defaultOrder: 'asc' },
        ]}
        pollInterval={5000}
        emptyMessage="No open markets available"
      >
        {(marketId) => (
          <>
            <MarketCharts marketId={marketId} views={['consensus', 'distribution']} />
            <TradePanel marketId={marketId} />
            <PositionTable marketId={marketId} />
          </>
        )}
      </MarketOverlay>
    </FunctionSpaceProvider>
  );
}
```

**Example: Without filter bar**

```tsx
import { FunctionSpaceProvider } from '@functionspace/react';
import { MarketOverlay, MarketCharts, TradePanel } from '@functionspace/ui';

function App() {
  return (
    <FunctionSpaceProvider config={config} theme="fs-dark">
      <MarketOverlay state="open" showFilterBar={false}>
        {(marketId) => (
          <>
            <MarketCharts marketId={marketId} />
            <TradePanel marketId={marketId} />
          </>
        )}
      </MarketOverlay>
    </FunctionSpaceProvider>
  );
}
```

**Example: Custom composition with auth and stats**

```tsx
import { FunctionSpaceProvider } from '@functionspace/react';
import { MarketOverlay, AuthWidget, MarketStats, ConsensusChart, ShapeCutter } from '@functionspace/ui';

function App() {
  return (
    <FunctionSpaceProvider config={config} theme="fs-dark">
      <AuthWidget />
      <MarketOverlay state="open" searchPlaceholder="Find a market..." filterBarMaxWidth="800px">
        {(marketId) => (
          <>
            <MarketStats marketId={marketId} />
            <ConsensusChart marketId={marketId} height={400} zoomable />
            <ShapeCutter marketId={marketId} />
          </>
        )}
      </MarketOverlay>
    </FunctionSpaceProvider>
  );
}
```

**Layout swapping:** If you need full-page navigation instead of an overlay panel, use the state-driven or route-driven patterns described in [Composition and Usage](../composition-and-usage#market-discovery-patterns) with `MarketList` directly. For standalone filter bar usage, see [MarketFilterBar](./marketfilterbar) with `useMarketFilters`.

**Related:** `MarketFilterBar` (filter UI rendered internally), `MarketList` (list component used internally), `MarketCard` (individual card), `useMarketFilters` (filter state hook used internally), `useMarkets` (data hook for market listing)
