# PasswordlessAuthWidget

**`PasswordlessAuthWidget`**

A streamlined authentication widget that supports passwordless login and auto-signup with just a username. Also provides admin login and admin signup flows for password-protected accounts.

```tsx
import { PasswordlessAuthWidget } from '@functionspace/ui';
```

<WidgetDemo title="Live Demo" maxWidth="350px">
  <PasswordlessAuthWidget />
</WidgetDemo>

:::note
Authentication is disabled in the docs environment. The demo above shows the widget UI in read-only mode.
:::

**CSS class:** `fs-passwordless-auth`

**Props:**

| Prop                | Type                                                      | Default | Description                                                        |
| ------------------- | --------------------------------------------------------- | ------- | ------------------------------------------------------------------ |
| `requireAccessCode` | `boolean`                                                 | `false` | Show an access code field on the admin signup form                 |
| `onLogin`           | `(user: UserProfile, action: 'login' \| 'signup') => void` | --      | Called after successful login or auto-signup. `action` indicates which occurred. |
| `onSignup`          | `(user: UserProfile) => void`                             | --      | Called after successful admin signup                                |
| `onLogout`          | `() => void`                                              | --      | Called after logout                                                |

**States:**

| State              | Renders                                                                                          | Transitions To                                                           |
| ------------------ | ------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------ |
| **Idle**           | "Sign In to Trade" text and "Sign In / Sign Up" button                                           | Passwordless modal                                                       |
| **Passwordless**   | Modal with username field, submit, cancel, "Admin Login" link                                    | Idle (cancel/success), Admin (link), Authenticated (success)             |
| **Admin**          | Modal with username + password fields, submit, cancel, "Back to Sign In / Sign Up" and "Create Admin Account" links | Idle (cancel/success), Passwordless (link), Admin Signup (link), Authenticated (success) |
| **Admin Signup**   | Modal with username + password + confirm password + optional access code, submit, cancel, "Back to Admin Login" link | Idle (cancel/success), Admin (link), Authenticated (success)             |
| **Authenticated**  | Wallet balance (`$X.XX`), username, "Sign Out" button                                            | Idle (sign out)                                                          |

**Behavior:**

* **Passwordless flow:** The user enters a username and submits. If the account exists (no password), they are logged in. If the username is new, an account is created automatically. If the account requires a password, an error message directs the user to Admin Login.
* **Admin login flow:** Standard username + password authentication. Accessible via the "Admin Login" link in the passwordless modal.
* **Admin signup flow:** Username + password + confirm password (+ optional access code). Accessible from the admin login view.
* **Silent re-auth integration:** When `showAdminLogin` becomes `true` on the auth context (triggered by `FunctionSpaceProvider` detecting a password-protected stored username), the widget auto-opens the admin login modal with the username pre-filled.
* **Form validation:**
  * Username validated via `validateUsername()` (3-32 chars, alphanumeric + `.` `-` `_`) on passwordless and admin signup submit.
  * Password minimum: 6 characters. Confirm password must match (admin signup only).
  * Access code field shown when `requireAccessCode` is `true` but is not enforced.
* **Loading UX:** All inputs and buttons are disabled during auth operations. Submit button text updates (e.g., "Signing in...", "Logging in...", "Creating account...").
* **Error handling:** The widget manages its own `formError` state from caught exceptions. `PASSWORD_REQUIRED` errors from the passwordless flow display a message directing users to Admin Login.
* **Modal behavior:** Escape key closes the modal. Clicking the backdrop closes the modal. All fields and errors clear on view transitions.

**Context interactions:**

* **Reads (via `useAuth`):** `user`, `isAuthenticated`, `loading`, `showAdminLogin`, `pendingAdminUsername`
* **Writes (via `useAuth`):** `passwordlessLogin()`, `login()`, `signup()`, `logout()`, `clearAdminLogin()`

**Internal calls:** `useAuth`, `validateUsername`, `PASSWORD_REQUIRED`

**Example:**

```tsx
<FunctionSpaceProvider config={config} theme="fs-dark">
  <PasswordlessAuthWidget />
</FunctionSpaceProvider>
```

```tsx
<PasswordlessAuthWidget
  requireAccessCode
  onLogin={(user, action) => {
    console.log(`${action}: ${user.username}`);
  }}
  onLogout={() => console.log('Signed out')}
/>
```

**Comparison with AuthWidget:**

| Feature                   | AuthWidget          | PasswordlessAuthWidget     |
| ------------------------- | ------------------- | -------------------------- |
| Default login flow        | Username + password | Username only (passwordless) |
| Auto-signup               | No                  | Yes (on new usernames)      |
| Password-protected fallback | N/A               | Admin Login modal           |
| Silent re-auth support    | No                  | Yes (auto-opens admin prompt) |

**Related:** `AuthWidget` | `useAuth` (hook) | `passwordlessLoginUser` (core) | `PASSWORD_REQUIRED` (constant) | `UserProfile` (type)
